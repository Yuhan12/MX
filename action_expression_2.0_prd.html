角色动作表情 2.0

社交基建 · 交互需求文档
v1.02026.06.10阶段：探索实验期
背景

角色动作表情是玩家在游戏中表达情绪、展现个性、深化社交关系的核心载体。

本次 2.0 迭代将动作从纯单人扩展到双人协作和多人群体两个维度，建立完整的触发、邀请、加入交互流程，并在特效、音效、文字气泡等表现层面进行探索。当前阶段以实验探索为主，首批制作 8 个动作验证可行性，为后续批量产出积累经验。

概

功能概述

制作出一组能够涵盖交往行为并能准确表达的表情动作：

  • 单人表意——让玩家通过非文字形式进行日常表达，内容丰富，从中挑选
  • 双人互动——亲昵动作表达双方关系，如拥抱、摸摸头
  • 多人欢乐——欢乐向或氛围向动作，让玩家低负担地参与进来

"准确表达"意味着纯动作可能不够，需要探索特效、音效、文字气泡等辅助手段的组合效果。

首批规模：单人 4 个、双人 2 个、多人 2 个，共 8 个动作进行实验。

子功能条目
  • 入口与动作面板：径向菜单触发，底部拉起动作选择面板，支持分类浏览
  • 单人动作：即点即播的情绪表达，移动或交互打断
  • 双人动作：两人配合完成，支持开放邀请（模式 A）和定向邀请（模式 B）
  • 多人动作 · 强关联：物理串联的群体互动（如排火车），领头带队、队员跟随
  • 多人动作 · 弱关联：氛围式群体互动（如篝火舞），各自独立、靠近即参与
  • 表现形式：动画 + 特效 / 音效 / 文字气泡的探索与组合，这部分内容随着内容的制作添加

1

入口与动作面板

一、功能概述

动作面板是浏览和触发所有动作的唯一入口。玩家点击自己角色呼出径向菜单，选择"表情"后从屏幕底部拉起动作面板。面板分单人 / 多人两个页签，网格展示动作图标与名称。

  • 径向菜单：多功能入口，"表情"是其中一项
  • 底部面板：承载动作浏览与触发，空间充足，支持增长
  • 分类页签：单人动作 / 多人动作
二、功能规则
入口规则
  1. 玩家点击自己的角色呼出径向菜单，菜单中包含"表情"入口。
  2. 点击"表情"后，径向菜单关闭，底部动作面板滑入。
  3. 点击面板外空白区域收起面板；点击其他交互时面板自动收起并打开对应交互。
  4. 角色处于双人 / 多人动作中时，不可呼出径向菜单，需先结束当前动作。
动作触发通用规则
  1. 仅展示玩家已解锁的动作，未解锁动作不出现在面板中。
  2. 点击动作图标即触发，无二次确认；触发后面板保持常驻，玩家可继续选择其他动作。
  3. 首次点击立即响应，后续点击在配置间隔内静默忽略（不排队、不提示）。间隔时长由策划配表，默认建议 0.5 秒。
  4. 同一时间只能播放一个动作
  5. 单人动作：点击即播，移动 / 交互 / 新动作立即打断。
  6. 双人 / 多人动作：触发后进入邀请或加入流程，面板收起。

触发节点触发表现

点击"表情"收起菜单，底部滑入动作面板

三、界面规则
径向菜单
  1. 布局：环形排列功能图标，"表情"图标高亮区分。
  2. 触发：点击自己角色弹出，点击菜单外收起。
  3. 锁定态：双人 / 多人动作中不可呼出。
底部动作面板

显示规则：

  1. 分为"单人动作"和"多人动作"两个页签，默认展示"单人动作"。
  2. 每个动作以图标 + 名称展示，目前按动作 ID 排序。
  3. 面板高度不遮挡角色主体视野。
  4. 动作数量超过一屏时，支持翻页浏览。
  5. 节流期间点击的动作图标无视觉反馈，不做置灰或 loading——保持轻量。

交互行为：

  • 点击页签切换动作列表
  • 点击动作图标触发流程，面板保持常驻，玩家可继续浏览或切换动作
  • 点击面板外空白区域收起面板；点击其他交互时面板自动收起并进入对应交互

关闭方式：玩家点击面板外空白区域收起面板；或点击其他交互时面板自动收起并打开对应交互内容。

空状态：该分类无已解锁动作时，显示空状态文案 “暂时还没有表情呢”

2

单人动作

一、功能概述

单人动作是玩家独立触发的情绪表达，不需要他人配合。点击即播放，移动或交互操作立即打断。

  • 即时触发：点击即播放，无确认步骤
  • 瞬时 / 循环：由动作资源决定——瞬时播完回 idle，循环持续到被打断
  • 无停止按钮：通过移动、交互或使用新动作打断
二、功能规则
触发规则
  1. 点击单人动作图标，角色立即播放对应动画。
  2. 同一时间只能播放一个动作，新动作覆盖旧动作。
  3. 播放时保持角色当前朝向，不做调整。
边界情况
  1. 连续快速点击不同动作：直接切换到最后一次点击的动作。
  2. 播放中收到双人动作邀请：可以收到邀请，接受后单人动作被打断。
三、界面规则
播放中界面
  1. 角色表现：播放动画，可能伴随特效 / 音效 / 文字气泡（由动作资源配置）。
  2. HUD：常规 HUD 保持显示。
  3. 其他玩家视角：可正常看到该角色的动作和特效表现。
3

双人动作

一、功能概述

双人动作需要两位玩家配合完成，发起方式由动作配置决定。参与方就位后同步播放，播放期间双方锁定，播完自动退出。

  • 模式 A — 开放邀请：发起者保持邀请姿势，头顶出现交互按钮，能看到按钮的玩家点击即加入
  • 模式 B — 定向邀请：发起者点击目标角色，在目标的交互上触发邀请，交互按钮仅被邀请方可见
  • 就位播放：如果参与者位置不对，自动移动到对应位置后同步播放
二、功能规则
模式 A — 开放邀请
  1. 点击模式 A 双人动作，角色进入邀请等待态，保持邀请姿势。
  2. 发起者头顶出现交互按钮，附近能看到按钮的玩家点击即可加入。
  3. 无超时限制，邀请态持续直到被打断。
  4. 发起者做任意交互操作即取消邀请（移动、打开功能等）。
  5. 有人加入后，若参与者位置不对，自动移动到对应位置后同步播放。
  6. 多人同时点击按钮：先到先得。
模式 B — 定向邀请
  1. 点击模式 B 双人动作后，点击目标角色选择邀请对象。
  2. 选中后，发起者头顶出现交互按钮，仅被邀请方可见。
  3. 无超时限制，只要双方在可视距离内按钮持续显示。
  4. 被邀请方点击按钮即接受，自动移动到位后同步播放。
  5. 被邀请方可忽略（不操作或走开）。
  6. 发起者做任意交互操作即取消邀请。
播放与退出
  1. 就位后同步播放完整动画。
  2. 播放期间双方都不可退出，直到动作播完。
  3. 播完后各自回到 idle。
触发条件表
触发节点触发条件触发表现
点击模式A动作已解锁面板收起，进入邀请姿势
点击模式B动作已解锁面板收起，进入目标选择
路人点击交互按钮模式A，在按钮可见范围路人移动到位，同步播放
目标点击交互按钮模式B，在可视距离内目标移动到位，同步播放
发起者任意交互邀请态中邀请取消，回 idle
边界情况
  1. 邀请期间发起者断线：邀请态自动取消。
  2. 被邀方正在单人动作中：可正常看到按钮，接受后打断单人动作。
  3. 被邀方在双人/多人动作中：不可被选中。
  4. 播放期间断线：另一方中断，显示"对方已离线"。
三、界面规则
邀请等待态

发起者：角色保持邀请姿势，头顶显示交互按钮；任意操作即取消。

其他玩家（模式A）：能看到按钮的玩家可点击加入。

被邀方（模式B）：仅被邀请方能看到发起者头顶的交互按钮。

播放中界面
  1. 双方同步播放动画。
  2. 操作锁定：移动和功能按钮不可用。
  3. 镜头不做特殊处理，保持当前视角。
  4. 播完后锁定解除。
4

多人动作 · 强关联

一、功能概述

强关联多人动作中，参与者物理串联，有明确的空间排列关系。以排火车为例：发起者成为火车头带队移动，后续玩家依次接入队尾跟随。

  • 发起：与双人动作一致，发起者头顶出现交互按钮
  • 加入：靠近并点击交互按钮，接入队尾
  • 行为锁定：参与者全程锁定，不可做其他操作
  • 领头退出：移动权限交给加入的第二个人，直到所有人退出
二、功能规则
发起与加入
  1. 点击多人强关联动作，成为队列领头，立即开始动作。
  2. 领头头顶出现交互按钮，附近玩家靠近后可点击加入。
  3. 触发距离与当前界面 UI 显示范围一致。
  4. 加入后固定接入队尾。
  5. 每个动作有独立的人数上限，由策划配表配置。队伍已满时按钮显示，玩家点击时提醒“队伍已满，稍后再试试吧~”。
行为锁定
  1. 参与者禁止：进入其他场景、打开功能界面、与物件交互。
  2. 尝试时提示"请先退出互动"。
  3. 必须先通过 UI 按钮主动退出才能做其他操作。
退出与解散
退出场景触发条件结果
领头退出点击退出按钮全队解散
中间成员退出点击退出按钮脱离，后方自动补位前移
队尾退出点击退出按钮脱离，队伍缩短
领头断线网络断开全队解散
成员断线网络断开脱离，自动补位
移动与碰撞
  1. 领头人控制队伍移动方向。
  2. 遇障碍物时领头人自动绕开，后方成员跟随领头路径。
  3. 突然出现的角色被轻微推开后回归队伍。
  4. 队伍到场景边缘：跟随领头可移动范围，不超出边界。
  5. 队伍不支持锁定，未满时始终可加入。
三、界面规则
各视角界面
视角界面表现
领头头顶显示队伍状态标签（如"火车 N/M 人"）；屏幕左下方显示退出按钮；其他功能按钮置灰
队员跟随领头行进；屏幕左下方显示退出按钮；其他功能按钮置灰
路人可看到队伍行进；靠近时出现"加入XX"按钮；队伍已满时按钮不显示
5

多人动作 · 弱关联

一、功能概述

弱关联多人动作中，参与者之间没有物理连接，各自独立表演同一动作。触发条件是关联的（靠近发起者），但动作本身不需要空间配合。以篝火舞为例：发起者开始跳舞，路人靠近后可选择加入一起跳，加入者在自己当前位置播放同款动作。

  • 无空间约束：参与者各自在原位播放动作，无需排列或对位
  • 低负担加入：靠近发起者看到按钮，点击即参与
  • 独立退出：任何人退出不影响其他人，发起者退出不解散
二、功能规则
发起与加入
  1. 点击弱关联多人动作，发起者在当前位置开始动作表演。
  2. 发起者头顶出现交互按钮，附近玩家靠近后可点击加入。
  3. 加入者在自己当前位置播放同款动作，不做位移。
  4. 人数上限由配表配置。
退出规则
  1. 参与者通过屏幕左下方退出按钮主动退出。
  2. 个人退出不影响其他参与者。
  3. 发起者退出不解散其他人——与强关联的核心区别。
  4. 行为锁定规则与强关联一致：参与中禁止场景切换、功能交互。
边界情况
  1. 所有人都退出：动作自然结束，无需特殊处理。
  2. 参与者断线：该玩家脱离，其他人不受影响。
  3. 场景中多个同类弱关联动作：各自独立，不互相干扰。
三、界面规则
各视角界面
视角界面表现
发起者播放动作，头顶显示标识；屏幕左下方退出按钮
参与者在自身位置播放同款动作；屏幕左下方退出按钮
路人可看到多人在跳同一动作；靠近发起者时出现加入按钮

表现形式

动作的识别度和表达力取决于多个表现层的组合。本阶段对各层进行探索，在首批 8 个动作中尝试不同组合，找到适合竖屏俯视角的最佳方案，也因此会逐步拓展功能。

表现层方向状态
动画核心层，由动作资源决定瞬时 / 循环、幅度、节奏基础
特效两个方向均会尝试：角色附着（心心冒出、光环环绕）和场景落点（脚下光圈、地面标记）。特效与人物的关系、位置、规模待首批试制确定探索中
音效仅自己可听，不广播给其他玩家。场景中大量玩家同时使用动作时避免音效叠加造成混乱已确认
文字气泡头顶文字气泡，固定文案（由动作资源配置），不支持玩家自定义已确认
识别度验证：音效仅自己可听，意味着其他玩家感知动作完全依赖视觉通道（动画 + 特效 + 气泡）。首批试制时建议有意识地测试：纯动画 vs 动画+特效 vs 动画+特效+气泡，找到"让别人注意到你在表达"的最低有效组合。

标注但暂不设计

  • 动作获取渠道与使用门槛随各类规划逐步增加
  • 新动作获取提示（红点 / 弹窗）后续补充
  • 环境 / 场景 / 天气使用门槛暂不考虑
  • 动作详情页 / 动画预览暂不考虑
  • "收藏 / 常用"快捷位记入待办，本版不处理
